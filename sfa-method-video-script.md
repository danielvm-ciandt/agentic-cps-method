# SFA Method — 1-Minute Video Script + Slide Outline

**Format:** screencast or slide-based narration  
**Target tool:** Loom / Synthesia / HeyGen / Descript  
**Pace:** ~150 words · 60 seconds

---

## Slide-by-Slide Script

---

### Slide 1 — Title (0:00–0:05)

**Visual:** "Spec-First Agile" logotype + tagline

**Narration:**
> Meet Spec-First Agile — a software delivery method that combines
> GSD's execution discipline, lean-spec's minimalism,
> and full Agile Practice Guide compliance.

---

### Slide 2 — Three commands (0:05–0:15)

**Visual:** Three boxes side by side:
`/initiating` → `/executing N` → `/closing`

**Narration:**
> The entire project runs through three commands.
> Initiating sets up your charter, backlog, and release plan.
> Executing runs your iterations. Closing handles UAT and project sign-off.
> One command per phase. The AI guides every step.

---

### Slide 3 — Waves and iterations (0:15–0:25)

**Visual:** Nested diagram — Project → Wave 1 (iter 1, iter 2) → Wave 2 (iter 3, 4, 5)

**Narration:**
> Iterations are grouped into waves — each wave delivers
> a coherent business goal. At the end of initiating,
> every iteration plan is already created as a stub.
> No blank pages when execution starts.

---

### Slide 4 — The story loop (0:25–0:40)

**Visual:** Linear flow:
`branch → discuss → spec → plan → execute → test → review-code → accept`

**Narration:**
> Inside each iteration, every story follows the same enforced loop.
> Branch first. Discuss before speccing. Spec before coding.
> Every acceptance criterion is scored for ambiguity — nothing above 0.20 gets through.
> Tests are generated from the spec. Code review is a gate, not a suggestion.
> The AI blocks any step that runs out of order.

---

### Slide 5 — Ceremonies = reports (0:40–0:50)

**Visual:** Table — Ceremony | Command | Report

- Iteration Planning → iter-N-plan.md
- Iteration Review → iter-N-review.md
- Retrospective → iter-N-retro.md
- Wave Review → wave-W-review.md

**Narration:**
> Ceremonies are not meetings. They're commands that write PMBOK 6-compliant reports.
> Run the command, the report is done.

---

### Slide 6 — Close (0:50–1:00)

**Visual:** "Spec-First Agile" + three principles as bullet points

**Narration:**
> Spec-First Agile. One file per artifact. One command per ceremony.
> Spec gates everything.
> Built for teams where the AI does the coding
> and humans own the decisions.

---

## Production Notes

- Total word count: ~148 words
- Recommended voice pace: measured, not rushed — pause 0.5s between slides
- Slides 4 and 5 benefit from animation (steps appearing one by one)
- Background music: none, or very low ambient (distraction risk with technical content)
- Captions: recommended — dense terminology
