Create `skills/acps-deliverable/SKILL.md`:

```markdown
---
name: acps-deliverable
description: 'Runs the deliverable workflow: selects the appropriate deliverable template, fills it with current project data, and presents it for human review before client submission.'
---

Follow the instructions in ./workflow.md.
```

Create `skills/acps-deliverable/workflow.md` with 5 XML steps:

**Step 1: Load config and select deliverable type** — read `.acps-config.json` for `planningDir`; fall back to `.planning`. Ask: "Which deliverable are you generating?"

```
1. Setup Deliverable — end of setup phase
2. Iteration Report  — end of iteration
3. Value Activation Deliverable — value activation phase
4. Roadmap Document — delivery roadmap
```

Map selection to template path:
- 1 → `templates/deliverables/setup-deliverable.md`
- 2 → `templates/deliverables/iteration-report.md`
- 3 → `templates/deliverables/value-activation-deliverable.md`
- 4 → `templates/deliverables/roadmap.md`

**Step 2: Gather data from planning artifacts** — read the following files to auto-populate deliverable sections:
- `{planningDir}/backlog.md` — iteration status, epic completion list
- `{planningDir}/change-log.md` — scope balance (BCP/FP)
- `{planningDir}/roadmap.md` (if exists) — delivery timeline
- Ask for any information not available in the above files: "What is the primary value delivered this {phase/iteration}?" and any template-specific fields that require human input.

**Step 3: Fill template and generate draft** — read the selected template file. Fill each section:
- Replace placeholder headings and instructions with actual project data
- Populate dates, iteration numbers, epic names, and completion status from backlog.md
- Fill scope section from change-log.md BCP/FP values
- Use current ISO date for document date
Write the filled document to `{planningDir}/deliverables/{deliverable-type}-{YYYY-MM-DD}.md` (create `deliverables/` directory if it does not exist).

**Step 4: Present for human review** — output the full generated document as a markdown block for human review. Output:

```
**Draft Deliverable Ready — Review Required**

The deliverable has been written to: {output_path}

Please review the document above. When ready:
- Type 'approved' to confirm it is ready for client submission
- Type 'edit' followed by your corrections to revise
```

Wait for user response. If 'edit': apply corrections and re-display. Repeat until 'approved'.

**Step 5: Report completion** — output:

```
**acps-deliverable complete.**

Document: {output_path}
Status: Approved for client submission

The document is in {planningDir}/deliverables/ — share with your client.
```
