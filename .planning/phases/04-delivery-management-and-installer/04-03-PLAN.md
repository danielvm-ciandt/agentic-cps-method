Create `skills/acps-new-epic/workflow.md` with 4 XML steps:

**Step 1: Load config** — read `.acps-config.json` for `planningDir`; fall back to `.planning`. Ensure `{planningDir}/epics/` exists; create it if not.

**Step 2: Collect epic information** — ask:
- "Epic ID? (e.g., ep-m01, ep-m02 — use kebab-case)"
- "Epic title? (short descriptive title)"
- "Epic overview? (1-3 sentence description of what this epic delivers)"
- "Requirements to include? (comma-separated list, or press Enter to leave blank)"
- "Non-goals? (what this epic explicitly does NOT cover, or press Enter to leave blank)"
- "Acceptance criteria? (list the conditions for completion, one per line)"

**Step 3: Fill template and write file** — read `templates/specs/epic.md`. Fill each section with the collected information:
- Replace the `# Title` heading with `# {epic_id}: {epic_title}`
- Fill the Overview section with the epic overview text
- Fill the Requirements section with the requirements as a checklist (`- [ ] item`)
- Fill the Non-goals section with the non-goals list
- Fill the Acceptance Criteria section with the ACs as a checklist

Write the filled file to `{planningDir}/epics/{epic_id}.md`.

**Step 4: Report completion** — output:

```
**acps-new-epic complete.**

Epic created: {planningDir}/epics/{epic_id}.md
Title: {epic_title}

Next steps:
1. Run `acps-new-story` to add stories under this epic
2. Run `acps-backlog` to estimate and sequence all epics
```

Create `skills/acps-new-bug/SKILL.md`:

```markdown
---
name: acps-new-bug
description: 'Registers a new bug before fixing it. Fills templates/bugs/bug-report.md, links to the parent epic, writes to {planningDir}/bugs/, and assigns BCP credit. Run before running acps-bug-fix.'
---

Follow the instructions in ./workflow.md.
```

Create `skills/acps-new-bug/workflow.md` with 5 XML steps:

**Step 1: Load config** — read `.acps-config.json` for `planningDir`; fall back to `.planning`. Ensure `{planningDir}/bugs/` exists; create it if not.

**Step 2: Collect bug information** — ask:
- "Bug ID? (e.g., bug-001, bug-ep-m01-001 — use kebab-case)"
- "Bug title? (short descriptive title)"
- "Which epic does this bug belong to? (e.g., ep-m01)"
- "Which story triggered this bug? (story ID or 'none' if standalone)"
- "Bug description? (what went wrong)"
- "Steps to reproduce? (numbered steps)"
- "Expected behavior? (what should happen)"
- "Actual behavior? (what actually happens)"
- "BCP credit for fixing this bug? (integer — this is the BCP assigned for the fix work)"

**Step 3: Fill bug template and write file** — read `templates/bugs/bug-report.md`. Fill each section with the collected information:
- Set the bug ID and title heading
- Fill the Description, Steps to Reproduce, Expected, and Actual sections
- Set the epic link and story link
- Set `bcp_credit: {bcp}` in the front-matter or metadata section

Write the filled file to `{planningDir}/bugs/{bug_id}.md`.

**Step 4: Link to epic** — check if `{planningDir}/epics/{epic_id}.md` exists. If yes: add a line to the Bugs section of the epic file (or append if no Bugs section): `- [{bug_id}]({relative_path_to_bug_file}) — {bug_title}`. If epic file does not exist: output warning "Epic file not found at {planningDir}/epics/{epic_id}.md. Bug registered standalone."

**Step 5: Report completion** — output:

```
**acps-new-bug complete.**

Bug registered: {planningDir}/bugs/{bug_id}.md
Linked to epic: {epic_id}
BCP credit: {bcp}

Next step: Run `acps-bug-fix` with this bug ID to begin the fix workflow.
```
