- Rewrite ADR-002 completely. Old ADR-002 is "pnpm via corepack as the package manager". New ADR-002 must read:

```markdown
### ADR-002: npm as the package manager

**Decision:** Standard npm with `package-lock.json`. No corepack, no pnpm.

**Rationale:** npm is the default Node.js package manager. Using npm removes the
corepack setup step from CI and reduces configuration surface. `npm ci` provides
deterministic installs from the lockfile.

**Trade-off accepted:** No workspace features. This repo has a single package —
workspaces are not needed.
```

- ADR-004 in the old file says "The `@semantic-release/npm` plugin is intentionally excluded from `.releaserc.json`" — this is incorrect; the new .releaserc.json DOES include the plugin (with npmPublish: false). Update ADR-004:

```markdown
### ADR-004: No npm publish in v1

**Decision:** GitHub Releases only. `@semantic-release/npm` is included with
`npmPublish: false` — the package version is managed but not published to the
npm registry.

**Rationale:** v1 installs via `npx acps@latest` from the GitHub release. npm
publish is deferred to v2 when the installer is production-ready.
```
