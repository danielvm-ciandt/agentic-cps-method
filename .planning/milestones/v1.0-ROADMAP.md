# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-20
**Phases:** 1–5
**Total Plans:** 30

## Overview

The complete `acps` v1.0 MVP — an installable CPS methodology package delivering the CI&T CPS framework as Claude Code skills across 10 AI IDEs. Ships via `npx acps@latest` with a full interactive installer, 30 skills covering the entire CPS inner loop (branch → discuss → spec → plan → execute → homologate), delivery governance, scope management, and project intelligence.

## Milestone Summary

**Key Decisions:**

- No framework in method repo — skills are static Markdown+JSON; framework adds complexity with zero user value
- `acps` as npm package name — short, available, matches CLI feel
- Markdown-only spec backend for v1 — zero config, works everywhere; GitHub Issues/ADO/Jira deferred to v2+
- Copy from old repo when content exists — avoid rewriting what already works; old repo has docs/ and bin/
- semantic-release branch mode — stable releases on `main`; no snapshot/pre-release complexity for v1
- Init guards via .acps-config.json existence check — prevents re-initialization accidents
- sessions: [] empty array literal as canonical (not null, not omitted) in story.md YAML
- tokens in totals uses nested YAML sub-keys, not dot notation
- Only story.md gets YAML front-matter; epic.md is plain Markdown — no programmatic consumer
- epics/ directory: flat list of epic.md files (not subdirectory-per-epic) — simpler for v1

**Issues Resolved:**

- delegate field missing .md extension in acps-scan and acps-code-map (fixed in commit 6a64794)
- markdownlint-cli vs markdownlint-cli2 config format mismatch (resolved via .markdownlint-cli2.yaml)

**Issues Deferred:**

- 2 pending UAT scenarios in Phase 01 HUMAN-UAT.md
- 10 pending UAT scenarios in MILESTONE-UAT.md
- 5 VERIFICATION.md files with status: human_needed (human sign-off pending)
- REQUIREMENTS.md checkbox tracking not updated during execution (tracking gap, not implementation gap)

**Technical Debt Incurred:**

- `agentic-cps-web` SvelteKit cockpit deferred to separate repo post-v1.0
